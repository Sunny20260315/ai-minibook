# Hermes Agent 初学者完全指南

## 一、什么是 Hermes Agent？

**Hermes Agent** 是由 AI 实验室 Nous Research 开发的开源自我进化型 AI 智能体框架，于 2026 年 2 月正式开源，采用 MIT 许可协议。

它的 slogan 是“The agent that grows with you”——一个会随着你的使用而长大的 AI 助手。

与大多数 AI 聊天机器人和编程助手最大的区别在于：**Hermes 不只是被动执行指令，它会在与你的每一次交互中自动学习、创建技能、积累记忆，并持续优化自己的行为方式**。

用一句话概括：**Hermes 是一个“自带终身学习能力”的 AI 智能体**——它用得越久，越懂你，越能干。

### 1.1 核心理念：闭环学习系统

Hermes 最核心的突破在于它内置了一套完整的“**学习闭环**”（closed learning loop）。这个循环包括：

| 环节               | 说明                                                       |
| :----------------- | :--------------------------------------------------------- |
| **策划记忆**       | 每轮对话后主动提取重要信息存入 SQLite 数据库，而非全量存储 |
| **自主创建 Skill** | 完成复杂任务后自动将流程提炼为结构化技能文件               |
| **Skill 自改进**   | 根据反馈自动优化已有技能的执行步骤                         |
| **跨会话召回**     | 使用 FTS5 全文搜索 + LLM 摘要，在新对话中按需检索历史记忆  |

这套机制让 Hermes 能够在“执行 → 复盘 → 学习 → 进化”的循环中持续自我改进。

### 1.2 “三层记忆”系统

Hermes 设计了业界领先的 **三层记忆系统**，解决了大部分 AI 智能体“会话结束就遗忘”的痛点：

- **会话记忆**：当前对话的实时上下文，滚动压缩以控制 token 消耗，支持按需检索
- **持久记忆**：跨会话的用户偏好、项目背景等事实性信息，永久存储于 SQLite
- **Skill 记忆**：从经验中沉淀的程序性知识，即结构化技能文档，可随时调用

> **出处**：Hermes 橙皮书《Hermes Agent 从入门到精通》，由“花叔”编写，系统阐述了 Hermes 的三层记忆与 Skill 自进化机制。此外，官方 GitHub 仓库 README 也明确列出了关闭学习循环、自主技能创建、FTS5 搜索等核心特性。

### 1.3 Harness Engineering：Hermes 的思想根基

要真正理解 Hermes，必须了解一个概念——**Harness Engineering**（缰绳工程）。为了确保信息的准确性和一致性，这里的定义严格基于“橙皮书”和“Harness Engineering 系统解析”等已有文档的描述，不自行补充。

**什么是 Harness Engineering？**

Harness Engineering 是一套围绕 AI Agent 的信任与稳定执行的方法论。Martin Fowler 在 2026 年 4 月的文章中将 Harness Engineering 定义为 **一套围绕 coding agent 的信任建设模型**，核心是通过**上下文、约束、反馈回路和工程结构**，让人逐步敢于把任务交给 Agent。

用一个比喻来理解：模型是骑手，Agent 是马，Harness 就是马鞍、缰绳和围栏——没有 Harness，马也会跑，但跑向哪里无法确定。

**Hermes 如何践行 Harness Engineering？**

根据“橙皮书”的描述，“Hermes Agent 并非又一个普通的 Agent 工具，而是个将‘Harness Engineering’（缰绳工程）方法论**产品化**的、具备‘自改进’能力的 AI Agent。其核心特点是‘**出厂就带缰绳**’”——即内建了自动化学习循环、三层记忆系统、可自我进化的 Skill 系统等。

换句话说：OpenClaw 是一个需要你手动搭建的学习环境（工具箱），而 Hermes 是直接把 Harness Engineering 这套方法论做成了一个可以自动运行的产品。它的“出厂配置”就已经具备从实践中学习和进化所需的全部基础设施。

## 四、Hermes vs Claude Code vs OpenClaw 深度对比

这三个项目代表当今最火的三种 Agent 设计哲学。理解它们的区别，有助于选对适合自己场景的工具。

> **注意**：以下对比根据多个来源的综合信息整理，包括 Hugging Face 技术文章《深度对比：Hermes Agent vs OpenClaw vs Claude Code》、社区讨论和 GitHub 数据分析。

### 4.1 概览对比

| 对比维度       | Hermes Agent                               | Claude Code                    | OpenClaw                          |
| :------------- | :----------------------------------------- | :----------------------------- | :-------------------------------- |
| **开发者**     | Nous Research                              | Anthropic                      | 开源社区                          |
| **定位**       | 底层通用 Agent 运行时框架                  | IDE 集成开发代理               | 个人 AI 助理                      |

| **工具调用**   | 40+ 内置，MCP，六种执行后端                | 40+ 工具，ReAct 循环           | 六层权限模型，Unix 哲学           |

### 4.2 定位差异详解

#### Hermes Agent ——“自进化的 AI 运行时”

Hermes 的核心定位是 **Agent 本体的执行与学习引擎**。它不是一个简单的聊天机器人，也不是某个 IDE 的代码补全插件，而是把 agent 需要的基础能力（记忆、工具调用、错误恢复、平台接入）打包成一套可配置的框架。它的核心问题是：“怎么让 Agent 在长期使用之后，变得更稳、更熟练，也更懂你？”

**一句话**：**它干完活之后，会自己把流程保存成 Skill，下次直接复用**。

#### Claude Code ——“坐在你身边的高效编码专家”

由 Anthropic 推出的 **开发者向 CLI Agent**，深度聚焦代码能力，在 SWE-bench 评测中位列榜首。它在 React + Ink 终端界面上通过 ReAct 循环控制 40+ 编程工具，并已与 VS Code 和 JetBrains 集成，是开发场景中追求最高代码质量时的首选。

**一句话**：**你得坐在电脑前，盯着它在 IDE 里帮你高效写代码**。

#### OpenClaw ——“功能最全的开源个人 AI 助理”

这是当前 **GitHub 上最受欢迎的开源 Agent 项目之一**，GitHub 星标已突破 36 万，累计 5,000+ 次提交，300+ 贡献者，已集成 20+ 聊天平台、131 个扩展和 53 个社区技能。它的核心优势在于：极致定制化、极高的可玩性、社区技能生态极其成熟。结合 Gateway 和六层安全模型，你可以赋予它从消息调度到浏览器自动化的丰富能力。

**一句话**：**你可以用手机远程指挥它，但需要给它装 Skill、喂数据、手动训练它干活**。

### 4.3 Harness Engineering 演进的视角

还可以从 **Harness Engineering** 的演进来看三者的差异：

| Agent            | 在 Harness 演进中的位置                                 |
| :--------------- | :------------------------------------------------------ |
| **Claude Code**  | 焦点在编码场景，工程架构服务于交互式开发                |
| **OpenClaw**     | 提供了丰富的功能组件，但 Harness 设计仍需人类搭建和维护 |
| **Hermes Agent** | 将 Harness Engineering 方法论**产品化**，出厂即内建     |

用一句话总结三者的演进关系：

> **Claude Code** 你需要坐在电脑前盯着它干活；**OpenClaw** 你可以远程指挥，但还得手动给它装 Skill、喂数据、培养它；**Hermes** 又往前走了一步——它干完活之后，会自己把这套流程保存成 Skill，下次直接复用。

## 五、快速上手（5 分钟起步）

### 5.1 安装要求

- 操作系统：Linux、macOS、Windows WSL2、Android Termux
- Python 3.10+
- 建议内存 4GB+（如需本地跑模型需 16GB+）

### 5.2 一行命令安装

在 Mac 或 Linux 终端执行：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

> **出处**：官方安装脚本，Hermes Agent 官方命令行安装指南。

> **提示**：如需中文资料，可查阅“Hermes Agent 安装与部署指南：一步步教你如何使用‘爱马仕Agent’（附飞书接入教程）”，其中详细涵盖了一键安装步骤、六种执行后端选择和飞书接入教程。

### 5.3 配置模型

```bash
hermes setup
```

按向导配置：

- 选择 LLM 提供商（OpenRouter / OpenAI / Anthropic / Ollama）
- 输入 API Key
- 选择模型

### 5.4 启动

```bash
hermes
```

### 5.5 核心速查命令

首次使用即可尝试这些命令开启你的智能体之旅：

- `/ask`：普通对话
- `/skills`：查看已生成的技能
- `/mem`：手动写入持久记忆
- `/model use [模型名]`：切换模型
- `hermes gateway setup`：配置微信/飞书/Telegram 等
- `hermes claw migrate`：从 OpenClaw 一键迁移

## 六、总结

**Hermes Agent 的本质是一个“自带终身学习能力的 AI 智能体运行时”**。

它不是普通聊天机器人，不是 IDE 补全工具，也不是 OpenClaw 的克隆版。它的核心区别在于 **内置了完整的学习闭环**——Auto 记忆、Auto Skill、Auto 进化、Auto 优化。

如果你希望有一个 **24×7 全天候为你服务、越用越顺手、能自己总结套路和经验** 的 AI 助手，Hermes 是目前最具前瞻性和务实性的开源框架之一。正如社区所总结的：“OpenClaw 是工具箱，Hermes 是员工，Harness Engineering 是怎么培训员工的手册”。
